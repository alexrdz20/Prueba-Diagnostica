PROBLEMA 1: Polinomio (x+1)^n usando Triángulo de Pascal
====================================================================

1. Captura de Variables y El Problema del Desbordamiento
Al calcular el Triángulo de Pascal hasta la fila n=100, los números se vuelven gigantescos. Python lo maneja automáticamente, pero en Java los tipos nativos (int/long) no son suficientes, por lo que usamos la clase BigInteger. Primero, pedimos al usuario los valores de n y x.

[Fragmento en Python - Menú de entrada]
    # Entrada para n
    while True:
        entrada_n = input("Ingrese el valor de n (Enter para 100): ").strip()
        if not entrada_n:
            n = 100
            break
        try:
            n = int(entrada_n)
            if n >= 0: break
            print("Error: n debe ser un número no negativo.")
        except ValueError:
            print("Error: Entrada inválida. Ingrese un número entero.")

[Fragmento en Java - Menú de entrada]
        Scanner sc = new Scanner(System.in);
        // Entrada para n
        while (true) {
            System.out.print("Ingrese el valor de n (Enter para 100): ");
            String input = sc.nextLine().trim();
            if (input.isEmpty()) break;
            try {
                n = Integer.parseInt(input);
                if (n >= 0) break;
                System.out.println("Error: n debe ser un numero no negativo.");
            } catch (NumberFormatException e) {
                System.out.println("Error: Entrada invalida. Ingrese un numero entero.");
            }
        }

2. Generación del Triángulo de Pascal
Para obtener los coeficientes sin fórmulas factoriales, utilizamos una lista dinámica (List en Python, ArrayList en Java). Comienza con la fila [1]. La siguiente fila se genera colocando un 1 al inicio, sumando los adyacentes, y terminando con otro 1.

[Fragmento en Python - Triángulo]
def generar_fila_pascal(n):
    fila = [1]
    for _ in range(n):
        nueva_fila = [1]
        for i in range(len(fila) - 1):
            nueva_fila.append(fila[i] + fila[i+1])
        nueva_fila.append(1)
        fila = nueva_fila
    return fila

[Fragmento en Java - Triángulo]
        List<BigInteger> fila = new ArrayList<>();
        fila.add(BigInteger.ONE);

            List<BigInteger> nueva = new ArrayList<>();

                nueva.add(fila.get(j).add(fila.get(j + 1)));
            }
            nueva.add(BigInteger.ONE);
            fila = nueva;
        }

3. Formateo y Evaluación del Polinomio
Recorremos los coeficientes. La potencia más alta se asigna al primer término y disminuye. Al mismo tiempo calculamos "Coeficiente * (x^potencia)" para sumar el total.

[Fragmento en Python - Formateo y cálculo]
    resultado = 0
    pasos = []
    for i, coeff in enumerate(coeficientes):
        potencia = n - i
        valor_termino = coeff * (x ** potencia)
        resultado += valor_termino
        pasos.append(f"{coeff}*({x}^{potencia}) = {valor_termino}")

PROBLEMA 2: Validador de Notación FEN
====================================================================

[Fragmento en Python]

2. Campo 1: Ubicación de las Piezas
Se separa el primer campo por '/' para obtener las 8 filas. Cada fila debe sumar exactamente 8 casillas (los dígitos suman espacios vacíos, las letras suman 1).

[Fragmento en Python]
    # 2. Validar Piece Placement
    ranks = placement.split('/')
    if len(ranks) != 8:
        return False
    for rank in ranks:
        squares = 0
        for char in rank:
            if char.isdigit():
                if char == '0': return False
                squares += int(char)
            elif char.lower() in 'pnbrqk':
                squares += 1
            else:
                return False
        if squares != 8:
            return False

---

3. Campos de Juego (Turno, Enroque, Peón al paso)
El color debe ser 'w' o 'b'. El enroque solo acepta caracteres válidos (sin repetir) mediante RegEx. El peón al paso verifica coordenadas a-h y 3-6.

[Fragmento en Python]

    if color not in ['w', 'b']: return False

    # 4. Validar Castling

            return False

PROBLEMA 3: Traductor de Palabras Reservadas
====================================================================

1. El Diccionario y Reserva Dinámica de Memoria
El código C necesita un diccionario (struct) para comparar las palabras. Luego usa malloc para cargar todo el archivo fuente a la memoria en un solo bloque, en lugar de leerlo lentamente línea por línea.

[Fragmento en C - Estructura]
typedef struct {
    const char *eng;
    const char *esp;
} PalabraReservada;

PalabraReservada diccionario[] = {
    {"auto", "automatico"}, {"break", "romper"}, {"while", "mientras"} // ...
};

[Fragmento en C - Malloc]
    /* Obtener el tamaño del archivo y reservar memoria dinámicamente */

    tamano = ftell(archivo);
    fseek(archivo, 0, SEEK_SET);

    codigo = (char *)malloc(tamano + 1);

3. Análisis Léxico (Extracción de Palabras)
Si no estamos en un comentario ni string, extraemos letras y las metemos en un arreglo "temporal". Cuando el código detecta un símbolo separador (ej. espacio, paréntesis), toma esa palabra y la traduce usando la función buscar_traduccion().
